Who is this Guide for?

It's for people who have downloaded and built Ublu, read enough of the Ublu Reference to know how to launch Ublu, and now want an introduction to using Ublu.

Why Ublu?

I wrote Ublu because I wanted a language to run on OpenBSD/Mac/Linux/Windows that could do systems programming on  IBM i Series OS ®.

I was supporting clients by writing individual programs using JTOpen which I have used since the 1990's to control the AS/400 and decided to consolidate.

I also had written a bridge in Java to IBM z/VM ® System Management API (SMAPI) called PigIron and now provide support in Ublu for SMAPI via PigIron.

How does Ublu work?

I made up a language and coded it in Java on top of other open source libraries. It sort of makes sense. I could have used extant language frameworks but I didn't want to spend my time understanding someone else's framework.

Ublu is easy to use, handsome but not beautiful, easy to understand and read quickly. It looks like a scripting language but has some advanced language features such as tasking, debugging and a TCP/IP port server mode.

For a formal summary of everything, see the Ublu Reference. That Reference tells you how to install and run Ublu and lists the Open Source project dependencies along with every command built into Ublu.
